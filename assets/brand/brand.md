# HuddleUp Brand

> Visual identity for the HuddleUp project. Use these assets verbatim — do not redraw, recolor, or restyle the mark.

## Logo

The HuddleUp mark is **Hex Huddle**: six teammates arranged in a hexagonal ring around a central huddle point, with a faint hex outline tying them together. It reads as a top-down view of a team meeting.

### Why this mark

- **Hexagon** = the strongest natural team-formation shape (honeycomb, sports huddle, table layout)
- **Six outer + one center** = teammates converging on shared work
- **Faded outline** = the boundary of the huddle without dominating the dots
- **Single color, flat geometry** = scales cleanly from 16px favicon to 1280px social preview

## Colors

| Token | Hex | Use |
|---|---|---|
| **Indigo (Primary)** | `#6366F1` | Logo mark, CTAs, links, primary brand colour |
| **Indigo Light** | `#A5B4FC` | Wordmark accent on dark backgrounds |
| **Indigo Surface** | `#EEF2FF` | OG image / marketplace icon background |
| **Ink** | `#0F172A` | Wordmark on light backgrounds, body text |
| **Muted** | `#475569` | Taglines, secondary copy |
| **Subtle** | `#64748B` | Captions, badges, footers |

**Never** combine the mark with any other primary colour. Indigo is the only brand colour.

## Typography

| Role | Family | Weight | Usage |
|---|---|---|---|
| Wordmark | System UI (`-apple-system, 'Segoe UI', Inter, system-ui`) | 700–800 | Logo lockups |
| Headlines | Same as wordmark | 600–700 | Site headings, OG titles |
| Body | Same as wordmark | 400–500 | Documentation, paragraphs |
| Code / CLI | `ui-monospace, SFMono-Regular, 'Cascadia Code', Consolas` | 500 | Terminal output, code samples |

We do **not** ship a custom font. System UI stacks keep the OSS site fast and license-free.

## Wordmark Rule

"Huddle" is rendered in ink (`#0F172A`); "Up" is rendered in indigo (`#6366F1`). On dark backgrounds, "Huddle" is white and "Up" is indigo-light (`#A5B4FC`).

Always: one word, camel-case, no space, no hyphen — **HuddleUp**.

Never: `huddle up`, `Huddle-Up`, `HUDDLEUP`, `huddleup` in display contexts.

## Clear Space

Keep clear space around the mark equal to the radius of one outer dot. Don't crop into the hex outline or fade the dots further.

## Don'ts

- Don't recolor (no greens, no reds, no rainbow).
- Don't add shadows, gradients, bevels, glows.
- Don't rotate the hex (the flat side stays vertical).
- Don't stretch — uniform scale only.
- Don't replace the wordmark font with a script / serif / display face.
- Don't pair the mark with another logo without equal clear space.

## Exporting Raster Assets

The SVGs are the source. Raster outputs (`.png`, `.ico`) live alongside them and are regenerated by a script — never hand-edited.

### Regenerate (after editing any master SVG)

```bash
npm run build:assets
```

This runs [`scripts/build-raster-assets.mjs`](../../scripts/build-raster-assets.mjs) and produces:

| Output | Size | Source |
|---|---|---|
| `assets/brand/favicon.ico` | 16/32/48 multi-res | `favicon.svg` |
| `assets/brand/apple-touch-icon.png` | 180×180 | `favicon.svg` |
| `assets/brand/og-image.png` | 1200×630 | `og-image.svg` |
| `assets/brand/github-social.png` | 1280×640 | `github-social.svg` |
| `vscode-huddleup/media/marketplace-icon.png` | 128×128 | `vscode-marketplace-icon.svg` |

The script uses [`sharp`](https://sharp.pixelplumbing.com/) for SVG→PNG rendering and [`to-ico`](https://github.com/kevva/to-ico) for the multi-resolution favicon. Both are dev dependencies, not shipped in the published npm package.

### SVG content rules

Because `sharp` uses `librsvg` under the hood, SVGs must use **XML-standard entities only**. Use numeric entities (`&#183;`, `&#8217;`) instead of named HTML entities like `&middot;` or `&rsquo;` — those will fail to parse.
